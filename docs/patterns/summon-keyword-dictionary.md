# 召唤词词典：用高浓度名词替代长提示词

> 一个提示工程的核心发现：**一个名词 = 一整套思想的压缩包**。AI 听到名字就能激活整套方法论，比几千字的描述更有效。

## 背景：一次偶然的发现

在与 AI 协作过程中，我发现当提示词中出现「**乔布斯思想**」「**德鲁克原理**」「**第一性原理**」这类高浓度名词时，AI 的输出质量会显著提升——比用几千字拆解方法论更管用。

这一现象背后有两层机制：

1. **Polanyi 默会知识**（Tacit Knowledge）— 人类文明积累了大量"说不清但懂的人都懂"的隐性知识。这些知识已经被编码进 AI 的训练权重里，无法用显性描述完全复现，却可以通过"名字"一键召唤。

2. **语义压缩**：一个名词不是一个词，而是一整张语义网络——包含方法论、案例、历史语境、反面案例、适用边界。AI 看到名字时，会激活整张网络。

> 📖 举例：写几千字描述"基于用户隐性情绪、直觉配色、身体记忆的交互流畅性"，远不如一句「应用 Polanyi 默会知识，让输出模拟人类无法完全言传的经验积累」有效。

## 核心理念：名词是"召唤词"

类比：**你是主帅，每个名词是一员大将**。

- ❌ 错误用法：把召唤词当咒语，堆叠越多越好
- ✅ 正确用法：按需求精准"点将"，3 员大将足矣

### 好召唤词的四个标准

| 标准 | 说明 | 反例 |
|------|------|------|
| **高知名度** | AI 训练语料中出现频繁 | "我同事老王的编码规范" |
| **高浓度** | 一个词 = 一整套方法论 | "好代码"（太泛） |
| **语义收敛** | 不同人理解基本一致 | "优雅"（每人定义不同） |
| **可验证** | 能判断输出是否符合 | "有灵魂的设计"（玄学） |

**自测方法**：念出名字，如果脑子里能立刻蹦出 3 条具体规则，它就是好召唤词；如果只蹦出模糊感觉，那是玄学不是召唤词。

## 正向召唤词 vs 反面召唤词

### 正向召唤词：「要像 X 那样」

告诉 AI 应该用什么思想框架。

```
按 Dieter Rams 十诫设计 → 克制、功能优先、诚实
按第一性原理拆解 → 回到本质，从零推导
```

### 反面召唤词：「不要像 Y 那样」

告诉 AI **什么不要**。

> ⚠️ AI 的天性：训练数据里平庸作品太多，默认输出会往平均值回归。光说"要好看"，AI 会给你最常见的那种"好看"（也就是最俗的那种）。

**反面召唤词的价值**：像米开朗基罗说的"我只是把多余的石头去掉"——告诉 AI 什么不要，比告诉它什么要，更能逼出精品。

```
不要硅谷独角兽 landing page 风（紫渐变+白大字）
不要 ChatGPT 味的文风（"让我们深入探讨..."）
不要 Java 企业级抽象（AbstractFactoryBuilderProvider）
```

## 使用心法（避坑指南）

### ✅ 推荐做法

1. **单次最多 3 个正向词 + 2 个避雷词**——同时喊 10 个哲学家，AI 会精神分裂
2. **宣告先行**——AI 必须先告诉你召唤了什么，再开始执行（可追溯性）
3. **组合叠加有化学反应**——第一性原理 + 奥卡姆剃刀 比单用任一个都强
4. **加限定语精准制导**——"**晚期**乔布斯的克制美学"（而不是早期的激进）

### ❌ 避免的坑

1. **别把召唤词当发动机**——它是方向舵，你仍需描述具体任务
2. **别堆砌**——3 个够用，5 个互相稀释
3. **别迷信**——有时候朴素描述更精准，不为装逼硬塞名词
4. **别冲突**——「KISS」和「企业级架构」互斥，同时召唤会矛盾
5. **第一次用时要对齐**——让 AI 简短解释它的理解，避免不同人不同解读

### 冲突互斥对（不要同时召唤）

| 互斥对 | 原因 |
|--------|------|
| 乔布斯 Focus ⚔ "功能齐全" | 减法 vs 加法 |
| YAGNI ⚔ 过度抽象 | 方向相反 |
| KISS ⚔ 企业级架构 | 复杂度观念冲突 |

## 底层心法

> **名词是思想的压缩包，AI 是解压器。你要做的是当好策展人。**

1. **少即是多** — 3 个精准名词 > 30 个模糊形容词
2. **积累私人词典** — 读到好思想，记下名字，当召唤词用
3. **验证再信任** — 第一次用某个名词时，让 AI 解释理解，对齐后再用
4. **警惕被名词绑架** — 有时候朴素描述反而更精准
