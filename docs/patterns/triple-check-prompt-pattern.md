# 三遍自纠提示词模式

## 适用场景

高复杂度任务、边界情况多、要求"一次到位"的场景：

- 按设计稿/规范代码实现复杂 UI 或组件
- 重构核心模块（类型定义、上传器接口、数据库层等）
- 涉及多状态的功能（loading / error / empty / success / 边界态）
- 数据处理、迁移、大数据量场景
- 任何"不能漏、不能错"的生产代码

**不适合的场景**见文末。

---

## 模式说明

### 为什么 one-shot 经常漏 edge case

LLM 默认采用**贪心解码**——每次生成挑最可能的下一个 token。这意味着：

- **首次输出 = 训练数据中最常见的解法**
- 边界情况、罕见场景 = 训练样本少 = 默认不会主动想到
- 没有外部反馈，模型不会自己"醒过来"说"哎我是不是漏了啥"

**结论**：一次回答 = 学生考试答题，写完就交卷。这种默认行为必然会漏掉非主路径的场景。

### 模式的核心思想

通过提示词显式喂给 AI 一个审慎工作流：

```
按需求实现 → 自行纠正三遍 →
模拟几万条数据测试 → 展示每轮反思 →
最终汇总对比
```

本质是：**把资深工程师的内心独白翻译出来，强制 AI 跑一遍**。

---

## 核心机制（四要素）

每一条都在对抗 LLM 的某个默认行为：

| 要素 | 打破的默认行为 | 本质作用 |
|------|--------------|---------|
| **自行纠正三遍** | 一次输出就交卷 | 强制"生成→批评→修订"循环，第 2、3 遍的修正是 one-shot 永远到不了的 |
| **模拟几万条数据测试** | 只考虑 happy path | 数量级锚定，激活压力测试心智，逼模型列举 edge case |
| **每次反思都要看到** | 偷懒跳过推理直接给答案 | 把"思考"从 hidden 变成 visible，AI 没法装思考 |
| **最终汇总对比** | 丢失中间过程价值 | 让用户看到改进幅度，模型自己也"见到"前后差距 |

### 要素详解

**1. 自行纠正三遍**

每一遍都是独立的视角：

- Round 1：完成主路径
- Round 2：压测边界，攻击自己的产出
- Round 3：规范审查，对照团队风格和项目约定

**2. 数量级锚定**

你说"测试一下"→ AI 测 3 条 happy path。
你说"几万条"→ AI 被迫思考：
- 性能怎样？（大数据量）
- 内存够吗？（OOM 风险）
- 边界分布？（空、最大值、重复、乱序、非法字符）
- 并发安全？（多任务同时跑）

**数量级强迫模型切换到规模化思维，自动带出性能和边界场景。**

**3. 每轮反思可见**

这条最反直觉但最重要。LLM 有个坏习惯：**假装思考**——嘴上说"让我仔细想想"，实际还是输出第一反应。

要求输出反思过程后：
- 模型必须真的生成具体的批评文本
- 这些文本会进入下一轮的 context，真的影响下一轮输出
- 用户能当场审查，发现"这轮反思没抓到点"时及时打断

**4. 汇总对比**

让改进幅度可审查，也让模型自己对三轮差异有"自我认知"。

---

## 为什么是"三遍"而不是其他

- **1 遍**：和 one-shot 没区别
- **2 遍**：足以抓到明显 bug，但批评视角单一
- **3 遍**：第 3 遍被迫从"新角度"审视（风格/规范/性能），边际收益最大
- **4+ 遍**：AI 开始自我重复或钻牛角尖，ROI 急剧下降

这和人类 code review 的经验一致——超过 3 轮 review 通常是沟通问题，不是代码问题。

---

## 与资深工程师内心独白的映射

资深工程师写代码时的内心独白：

1. 写一遍 → "这大概能跑"
2. 压测心智模型 → "如果输入是空呢？如果并发呢？如果数据有几万条呢？"
3. 按团队规范过一遍 → "代码风格/命名/注释 OK 吗？符合 CLAUDE.md 约定吗？"

这个模式就是把这份独白固化成可复用的提示词流程。

---

## 何时不该用

| 场景 | 原因 |
|------|------|
| 纯探索（"XX 是怎么工作的"） | 没有可验证的"正确答案" |
| 单行 typo 修复 | 杀鸡用牛刀 |
| 需求本身没想清楚 | 先澄清，再自纠（否则越纠越偏） |
| 紧急 hotfix | 优先解决问题，不求完美 |
| 上下文窗口吃紧 | 三轮输出会挤掉关键信息 |

---

## 弱点与风险

⚠️ **不是银弹**：
- 需求模糊时，三遍自纠只会越纠越偏
- 任务超出模型能力时，纠 10 遍也没用
- Token 消耗 ~3-5x 普通任务

⚠️ **成瘾风险**：
- 用户习惯"反正 AI 会自纠"后，可能懒得自己想清楚需求
- 把用户的思考外包给 AI 是危险的

**权衡**：节省的是**用户的返工精力**，代价是**模型的 token**。对注意力有限的开发者来说，这笔交易通常很划算。
